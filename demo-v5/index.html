Workflow n8n d'estimation automatisée
+ conformité Loi 25

Reçoit le payload du panier (v3) + des options (v4), valide en strict, génère un devis PDF via Carbone.io, le stocke dans Supabase Canada-Central, score le lead, pousse dans Airtable, et notifie Slack si Score > 200. Toutes les données restent au Québec. URL signée 30 jours. Rétention auditable.

1. Schéma logique des nœuds

Flux d'exécution séquentiel avec routage d'erreur explicite à chaque étape.

┌───────────────────────────────────────────────────────────────────────┐
│  ① Webhook POST /estimation-mark-systems                              │
│     ↓ payload JSON brut                                               │
│  ② Code Node : VALIDATE_AND_SCORE                                     │
│     • Schéma : workflows[], options{}, totaux{}, client{}             │
│     • Score = (h_mois × 10) + (prix_haut / 1000)                      │
│     • Si invalide → ④ ErrorTrigger → réponse 400                      │
│     ↓ payload normalisé + lead.score                                  │
│  ③ HTTP Request : POST Carbone.io (template DOCX → PDF)               │
│     ↓ binary PDF                                                      │
│  ④ Supabase Storage : upload devis-{lead_id}.pdf (bucket 'devis')     │
│     ↓ url_signed (TTL 30 jours)                                       │
│  ⑤ Supabase Insert : table 'leads' (upsert par email — idempotent)    │
│     ↓ lead_id                                                         │
│  ⑥ Airtable Upsert : base CRM (match email, sinon insert)             │
│     ↓                                                                 │
│  ⑦ Switch : Score > 200 ?                                              │
│     ├─ OUI → ⑧ Slack notif #leads-chaud (payload riche)               │
│     └─ NON → fin                                                      │
│  ⑨ Resend/Postmark : email transactionnel client (PDF + Calendly)     │
│     ↓                                                                 │
│  ⑩ Respond to Webhook : { status:'ok', lead_id, pdf_url, score }      │
└───────────────────────────────────────────────────────────────────────┘
       │
       └─→ Error Trigger global → table 'audit_errors' + Slack #ops

8. Registre Loi 25 (Québec)

    
    
      
  • Read-Only First. Le webhook reçoit, valide et calcule sans persistance avant consentement explicite (champ consentement_loi25).
    • 
      
  • Hébergement. Supabase région ca-central-1 (Montréal). Airtable workspace verrouillé Canada. Carbone.io self-hosted Docker au Québec.
    • 
      
  • Rétention. URL signée PDF = TTL 30 jours. Purge automatique via cron Supabase Edge Function.
    • 
      
  • Désinscription. Endpoint POST /loi25/forget — purge lead + audit + PDF dans < 24 h.
    • 
      
  • Traçabilité. Toute erreur loggée dans audit_errors avec PII masquées.
    • 
      
  • DPO. Contact public : dpo@mark-systems.ca.
